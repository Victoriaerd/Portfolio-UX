Design for an internal transcription tool

FreeComETT is an internal web application designed for FreeUKGenealogy. Volunteers use the tool to transcribe scanned historical documents records into structured digital data.

Role

UX Designer working as part of a multidisciplinary team including other UX designers, volunteer transcribers, and developers.

Challenges

  • Designing for accuracy-critical data entry
  • Supporting long, repetitive transcription sessions
  • Reducing cognitive load while viewing handwritten records
  • Providing clear validation and error feedback
  • Designing for accessibility and varied technical ability

Design Goals

The primary design goal was to minimise cognitive load and keep the user’s core task - transcription - continuously in focus, while supporting a complex legacy-to-modern workflow transition.

Tools

Figma, Photoshop, Miro, FigJam, Slack.


Design Principles & Rationale

To ground the design in technical and organisational realities, we interviewed the FreeComETT developers to understand the existing system architecture, data models, and constraints. This allowed to map the current and target information architecture, identify gaps in the existing interface, and align UX decisions with backend functionality.

The transcription tool already existed, but lacked a user-friendly front end. A key challenge was that many volunteers were long-term users of WinReg, a legacy desktop system that had reached end-of-support. Users were highly experienced, accustomed to established workflows, and initially resistant to moving to a browser-based tool.

This meant the design needed to:

  • Respect existing mental models from WinREG
  • Minimise disruption to expert workflows
  • Provide a familiar structure while introducing modern usability improvements

The initial post-login screen was intentionally simplified by deferring secondary information and actions to later stages of the workflow. The Transcribe action was made the dominant visual and interaction priority, while still preserving access to assignment creation for advanced users.

To improve discoverability and accessibility, key actions were moved from the footer into a persistent top navigation, with generous hit areas designed to support users with varying motor abilities and prolonged usage sessions.

The transcription workspace was structured to maximise usable screen space:

  • A document viewer occupying the majority of the viewport
  • A structured data-entry area aligned with the source document

Legacy System Transition Considerations

Many volunteer users had relied on the legacy WinREG system for years, creating strong mental models and workflow expectations. The design needed to minimise disruption by preserving familiar interaction patterns where possible, while still introducing a modern, scalable interface. This informed decisions around layout, terminology, and data entry structures to reduce the learning curve and resistance to change.

The interface was designed to keep scanned records and data entry fields visible simultaneously, reducing context switching. A table-based layout supports keyboard-heavy workflows, while inline validation helps prevent incorrect data formats.

Design Principles

1. Transcription first
The interface prioritises the primary task at every stage. Secondary information and actions are deferred until needed to reduce cognitive load and keep users focused during long transcription sessions.

2. Respect existing mental models
Many users had years of experience with a legacy desktop tool. Where possible, familiar interaction patterns and terminology were preserved to reduce friction and support adoption of the new system.

3. Design for sustained focus
The workspace supports extended, repetitive work by minimising visual noise, keeping the source document and data entry aligned, and avoiding unnecessary interruptions.

4. Prevent errors before they happen
Inline validation, structured inputs, and system feedback were prioritised to support accuracy in data entry and reduce downstream correction work.

5. Accessibility by default
Large clickable areas, clear hierarchy, and keyboard-friendly interactions were considered essential.

6. Align UX with system reality
Design decisions were informed by direct collaboration with developers and volunteers to ensure the interface reflected real system behaviour and could scale across different record types and workflows.

Research & Insights

Alongside developer interviews, I conducted interviews with volunteer transcribers to understand existing workflows, pain points, and expectations shaped by long-term use of the legacy WinReg system.

One consistent insight was that users were highly accustomed to tabular data entry. WinReg structured transcription as a table, allowing users to move quickly between fields using the keyboard. Early concepts explored alternative input structures, but user feedback clearly indicated that deviating from this pattern would slow users down and increase friction.

Based on this feedback, the transcription interface was designed around a table-based data entry model, preserving familiar interaction patterns while improving clarity, validation, and accessibility.

These interviews reinforced the importance of designing for continuity during the transition from a legacy system, particularly for users performing repetitive, accuracy-critical work.

User Journey Overview

The transcription experience spans multiple roles and system states. Mapping the end-to-end flow helped ensure the interface supported both individual transcription tasks and broader coordination workflows.

Primary Flow

1. Sign in
Users authenticate and land on a simplified entry point designed to orient them quickly around the primary action: transcription.

2. Assignment creation (Coordinator)
A coordinator creates an assignment and uploads source material (images, microfilm scans, or other media). Assignments may contain small batches or large volumes (up to hundreds of files).

3. Media upload & organisation
Uploaded files are organised into assignments and user-specific folders. Design considerations included:

  • Handling large batch uploads (up to 500 files)

  • Visibility and ownership of folders

  • Coordinator oversight versus individual user autonomy

4. Transcription task selection
Transcribers select an assignment or, in cases where they work from physical media, create their own task. This flexibility supports different volunteer workflows without forcing a single rigid model.

5. Transcription work
Users transcribe records (e.g. marriage, burial, death details) using a structured interface aligned with the source document.

6. Validation & error checking
Before submission, the system checks for formatting and data errors, surfacing issues early to reduce downstream correction.

7. Output & confirmation
Completed work is exported as a CSV file. Users receive confirmation via email once the upload is successful, reinforcing closure of the task.


Workspace & Interaction Design

Document Viewing & Image Controls

Historical source documents varied significantly in quality and legibility. To support accurate transcription, the document viewer includes image adjustment controls such as:

  • Zoom

  • Brightness and contrast

  • Sharpness

  • The ability to expand and collapse the image control panel as needed

These controls allow users to tailor the viewing experience to different document conditions without leaving the transcription context. Keeping adjustments lightweight and easily accessibleto help maintaining focus during long sessions.

In Load CSV files we grouped separately the CSV files on the FreeREG server and the CSV files in the FreeComETT account. The former may be very many, while the latter will always be few, so mixing them together and having to scroll to find the one from FreeComETT would have been cumbersome.

Customisable Data Entry


Transcription requirements vary significantly across record types. Some records contain a small number of fields, while others can require up to 67 individual fields, making a static data-entry layout impractical.

To address this, the transcription tool supports customisable column layouts, allowing users to tailor the data-entry table to their current task.

Users can:

  • Select which fields are visible in the table
  • Hide rarely used fields to reduce visual clutter
  • Reorder fields to match the structure of the source document
  • Save and reuse preferred layouts

This flexibility allows users to work efficiently without being overwhelmed by unnecessary inputs.



Accessibility

To support prolonged transcription sessions and diverse accessibility needs, users can further customise individual fields by:

  • Adjusting font size for readability
  • Applying colour cues to visually group or highlight fields
  • Expanding or collapsing configuration panels to maximise workspace

These controls allow users to adapt the interface to different document qualities, screen sizes, and personal preferences without disrupting the core workflow.


Auto-Copy & Efficiency Features

For records with repeated or shared values, an auto-copy option allows users to propagate data across fields automatically. This feature can be enabled or disabled per field, giving users control over when automation supports accuracy versus when manual entry is required.

This balance ensures efficiency gains without introducing unintended errors.

Summary and what's next

This work resulted in a production-ready UX now in active development, shaped by close collaboration with engineers and grounded in real transcription workflows. The design balances legacy familiarity with modern browser-based interaction, enabling long-tenured users to transition confidently while supporting scalability, accessibility, and power-user efficiency.


What's next

Run task-based usability testing with both experienced WinREG users and new transcribers to validate learnability, error rates, and time-to-completion across high-field-count records. Test alternative data-entry patterns (e.g. field grouping, or inline field entry).
